# TradTuneExplorer Search Database

The **TradTuneExplorer Search Database** is a PostgreSQL database designed for large-scale analysis and search of traditional music collections.

It powers the analytical backend of TradTuneExplorer, providing infrastructure for metadata search, melody matching, statistical analysis, recommendation algorithms, and exploration of relationships between tunes, recordings, artists, albums, countries, and session performance.

## Features

The database includes support for:

- Tune metadata management
- Recording, artist, album, and collection relationships
- ABC notation storage and normalization
- Melody fingerprint generation
- Interval, contour, rhythm, and n-gram analysis
- Fuzzy melodic matching
- Edit-distance similarity scoring
- Search-optimized materialized views
- Search ranking and relevance algorithms
- Tune popularity and statistical analytics
- Hidden gem detection and scoring
- Recording pathway analysis
- Artist transition analysis
- Tune transition networks
- Set signature generation
- Country popularity and similarity analysis
- Member activity analytics
- Recording search infrastructure
- High-performance analytical queries over hundreds of thousands of tune relationships

## Architecture

The database is designed as both a search engine and an analytical platform for traditional music.

Materialized views are used extensively to pre-compute complex relationships, allowing fast exploration of:

- Tune popularity
- Recording history
- Artist collaborations
- Tune transitions
- Artist pathways
- Hidden gems
- Melody similarity
- Recording pathways
- Country-level trends
- Member activity
- Statistical rankings

This architecture enables complex analytical queries to execute in milliseconds while operating over hundreds of thousands of musical relationships.

## Data

This repository intentionally contains **database structure only** and does **not** include:

- Tune records
- Recordings
- Settings
- Comments
- User data
- Exports from The Session

Users wishing to populate the database should obtain the data directly from **The Session** and comply with its license.
